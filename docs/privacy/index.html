Privacy Policy

Request Headers Chrome extension · Last updated: June 22, 2026

Request Headers (“the extension”) is a browser extension that lets you configure custom HTTP request headers for websites you choose. This policy explains what data the extension handles and how.

What we collect

The extension does not collect, transmit, or sell your data. There is no backend server, no analytics, and no account system.

Configuration you enter is stored locally on your device using Chrome’s chrome.storage.local API, including:

  • Per-site URL patterns
  • Header names and values you configure
  • Profile names and optional descriptions
  • Global and per-site on/off settings

Header values may include sensitive information (for example API keys or tokens) if you choose to enter them. You control that data; treat shared computers accordingly.

How data is used

Stored settings are used only to apply your header rules in the browser via Chrome’s declarative network request APIs on origins you explicitly configure and grant access to.

Data sharing

We do not share, sell, or transfer your data to third parties. Nothing leaves your device except normal browser traffic to sites you visit with your configured headers attached.

Data retention and deletion

Data remains on your device until you delete it in the extension, remove the extension, or clear extension data in Chrome settings (chrome://extensions → Request Headers → Remove or Clear data).

Permissions

The extension requests access to specific websites only when you enable header rules for those sites. Storage permission is used to save your configuration locally.

Children

The extension is not directed at children under 13.

Changes

We may update this policy. The “Last updated” date above will change when we do. Continued use after changes means you accept the updated policy.
